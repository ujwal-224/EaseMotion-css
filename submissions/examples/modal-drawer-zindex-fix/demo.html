ease-modal + ease-drawer — z-index Conflict Fix

Both components previously used hardcoded, overlapping z-index values (e.g. both at z-index: 1000), causing unpredictable stacking when a drawer was opened on top of a modal, or vice versa. This demo uses a shared z-index scale via CSS custom properties so layering is always predictable.

Click "Open Modal", then click "Open Drawer (from modal)" while the modal is open — the drawer should correctly layer ABOVE the modal and its backdrop, because both now read from the shared --ease-z-* scale instead of colliding on the same raw number.

Example Modal

This modal uses z-index: var(--ease-z-modal) for itself and var(--ease-z-modal-backdrop) for its backdrop.

Second Modal (Stacked)

Demonstrates a modal opened on top of another modal using --ease-z-modal-stacked, still above any open drawer.

Example Drawer

This drawer uses z-index: var(--ease-z-drawer), defined to sit above modal content but its own backdrop sits below the modal it was opened from when nested.
